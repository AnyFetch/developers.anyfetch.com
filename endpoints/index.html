Fetch API

Fetch API is designed to help you search in massive amounts of documents coming from various sources, in various formats.

Authentication

There are three ways to authenticate through Fetch API. Requests that require authentication will return 404 Not Found, instead of 403 Forbidden, in some places.

Basic authentication

    $ curl -H "Authorization: Basic HASH" http:http://api.anyfetch.com

Oauth authentication

    $ curl -H "Authorization: token OAUTH-TOKEN" http:http://api.anyfetch.com

Request parameter

    $ curl https:http://api.anyfetch.com/?access_token=OAUTH-TOKEN

##Client Error If request is not wellformed, this will result 400 Bad Request responce.

Hydraters

Hydrations endpoints, adding and improving original content of documents. Those endpoints are only available when the hydrater sending the request has been previously called on the document. You can't access them with an OAuth token, or using Basic Auth.

POST /hydraters/documents/{id}

Update document metadatas, datas and actions.

  • Response 204 SHOW
GET /hydraters/documents/{id}/file

Get the file associated with the document for hydration purposes.

Note: the file will be removed from Fetch API servers after hydration has completed.

  • Response 200 SHOW
