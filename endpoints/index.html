Fetch API Documentation

This page is anyFetch reference. See http://developers.anyfetch.com for informations regarding general usage.

Fetch API is designed to help you search in massive amounts of documents coming from various sources, in various formats. See authentication for authentication details.

Batch queryingGET /batch{?pages}

Queue multiple GET queries in a single request. For instance, you can call /batch?pages=/&pages=/document_types&pages=/providers to get one JSON object with all datas, indexed by url.

Status code will be 200 if all queries passed. If an error occured, the errored key will tell you which page failed to load.

Documents 

Access documents resources.

Search documentsGET /documents{?search, ?before, ?after,?document_type, ?token, ?_meta, ?has_meta, ?related_to, ?snippet_size, ?start, ?limit, ?no_hydration}

Search within all availables datas for documents matching specified filter.

Return aggregated informations computed over the result set. The score key indicated document’s relevance regarding query.

  • Parameters
  • search
    string (optional) Example: john smith

    Search query, probably the most important parameter for this query

    before
    date (optional) Example: 2014-03-21

    Only display documents created before this date.

    after
    date (optional) Example: 2014-01-25

    Only display documents created after this date.

    document_type
    array (optional) Example: 5252ce4ce4cfcd16f55cfa3f

    Only retrieve documents matching this document type. You can use the param multiple times to allow for multiples document_type.

    token
    array (optional) Example: 5252ce4ce4cfcd16f55cfa3f

    Only retrieve documents matching this token. You can use the param multiple times to allow for multiples token

    _meta
    string (optional) Example: John Smith

    Strict search on meta key. Replace meta with the name of the meta you wish to search on.

    has_meta
    boolean (optional) Example: 1

    Only returns document having the meta key. Replace meta with the name of the meta you wish to search on.

    related_to
    string (optional) Example: 53273bd31eab8a0e22c2e663

    Find documents related to the specified document

    snippet_size
    integer (optional) Example: 250

    Number of chars to include in the snippet

    start
    integer (optional) Example: 5

    0-based index of the first item to retrieve (for pagination).

    limit
    integer (optional) Example: 20

    Max number of items to retrieve (for pagination)
