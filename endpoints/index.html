Fetch API

GET /documents/{?start, ?limit, ?search, ?_meta, ?@meta, ?has_meta, ?related_to, ?document_type, ?snippet_size}
  • Parameters
  • _meta
    (required) string Strict search on `meta` key
    document_type
    (required) array Only retrieve documents matching this document type. You can use the param multiple times to allow for multiples `document_type`
    has_meta
    (required) string Only returns document having the `meta` key
    limit
    (required) integer Max number of items to retrieve (for pagination)
    related_to
    (required) string Find documents related to the specified document
    search
    (required) string Search query, probably the most important parameter for this query
    snippet_size
    Number of words in the snippet
    start
    (required) integer Index of the first item to retrieve (for pagination)
  • Response 204 SHOW

Hydraters

Hydrations endpoints, adding and improving original content of documents. Those endpoints are only available when the hydrater sending the request has been previously called on the document. You can't access them with an OAuth token, or using Basic Auth.

POST /hydraters/documents/{id}

Update document metadatas, datas and actions. For datas and metadatas, merging occurs on first level only. If the existing datas is {a:1, b:{c:2}, d:3} and you submit {a:2, b:{e:4}}, the resulting document will be {a:2, b:{e:4}, d:3}. To remove a key, simply set its value to null.

  • Response 204 SHOW
GET /hydraters/documents/{id}/file

Get the file associated with the document for hydration purposes.

Note: the file will be removed from Fetch API servers after hydration has completed.

  • Response 200 SHOW
